Stock Timing Radar v8.3 — Fundamental UI redesign + price target connection check

What changed:
1) Fundamental detail page redesigned to look closer to the requested UI.
   - Overview hero cards
   - Revenue / Net Income / EPS statement cards
   - Clearer estimate explanations
   - Linear price target rail

2) Fundamental table improved.
   - Added Prev Quarter / Year Ago fields for Revenue, Net Income, EPS
   - Added EPS Surprise %
   - Added Analyst count for price target

3) Price target connection logic hardened.
   - Yahoo quoteSummary now tries both query1 and query2
   - Price target connection status exposed in UI
   - Analyst count pulled when available
   - Above/below-current analyst split only shown when deterministically inferable from the consensus range; otherwise N/A to avoid fabricating counts

4) SEC data enrichment.
   - Prev quarter and year-ago values now carried through for core statement items

Notes:
- If Est. Revenue / Est. EPS shows N/A, it usually means no consensus estimate is available or Yahoo did not return the field.
- Per-analyst target-by-target split is not exposed by the free Yahoo summary endpoint, so the app does not fake those counts.
