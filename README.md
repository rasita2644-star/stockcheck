# Stock Timing Radar — V2.8 GitHub Pages Hybrid Static Mode

This version is designed for **GitHub-only deployment**:

- **Fundamental data** is static and generated by GitHub Actions daily/manual.
- **Technical data** is generated by GitHub Actions about every 15 minutes.
- **Analyst Consensus** uses Alpha Vantage BYOK: the user pastes their own API key in the browser, and the key is stored only in `localStorage`.
- No Python backend is required on GitHub Pages.

## Data layers

| Layer | File | Update method |
|---|---|---|
| Technical | `site/data/technical.json` | GitHub Actions every ~15 minutes |
| Fundamental | `site/data/fundamental.json` | GitHub Actions daily or manual |
| Compatibility | `site/data/scanner.json` | technical fallback / old UI compatibility |

The frontend merges `technical.json` and `fundamental.json` by ticker.

## GitHub Pages setup

1. Push this repo to GitHub.
2. Go to **Settings → Pages**.
3. Set **Source** to **GitHub Actions**.
4. Go to **Actions** and run:
   - `Update static fundamental data` once first.
   - `Update technical data and deploy GitHub Pages` after that.
5. The technical workflow will continue on a ~15-minute schedule.

## Important limitations

This is a static GitHub Pages setup, not a live Python server:

- Technical data is near-real-time, not tick-by-tick.
- Scheduled GitHub Actions can be delayed by GitHub.
- SEC/guidance data is precomputed, not refreshed from the browser.
- Alpha Vantage analyst consensus is manual BYOK and runs only when the user clicks Load.

## API key safety

Do **not** commit API keys.

This repo intentionally excludes:

- `.env`
- `alpha_vantage_api_key.txt`
- `.alpha_vantage_cache/`
- `.alpha_vantage_quota.json`

Alpha Vantage key handling in V2.8:

- The user pastes the key in the Analyst Consensus tab.
- The key is saved only in the browser's `localStorage`.
- The browser calls Alpha Vantage directly only when the user clicks `Load consensus`.
- The key is never written to project files.
